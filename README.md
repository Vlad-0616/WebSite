# Грузоперевозки.бел — Биржа грузоперевозок

## Описание проекта

**Грузоперевозки.БЕЛ** — это динамическая веб-платформа для прямого взаимодействия грузовладельцев и перевозчиков. Заказчики могут быстро разместить заявку на перевозку груза, а водители или транспортные компании — находить подходящие заказы и предлагать свои услуги. Платформа автоматизирует процесс поиска транспорта и отслеживания статуса перевозки, делая логистику прозрачной и эффективной.

## Целевая аудитория

| Сегмент | Описание |
|---------|----------|
| **Грузовладельцы** | Компании и частные лица, которым нужно перевезти грузы |
| **Перевозчики** | Владельцы грузового транспорта (ИП, компании, частные водители) |
| **Логисты и экспедиторы** | Специалисты, занимающиеся поиском транспорта под задачи клиентов |

---

## Ключевой функционал

- **Управление заказами:** размещение заказов на перевозку с указанием веса, габаритов, маршрута, типа погрузки, вида прицепа и стоимости
- **Управление транспортом:** добавление и редактирование транспортных средств перевозчиками
- **Поиск и фильтрация:** по направлениям, весу, типу кузова, дате загрузки, цене, типу загрузки
- **Отзывы и рейтинги:** формирование репутации участников
- **Личные кабинеты:** раздельные интерфейсы для заказчиков и перевозчиков
- **Админ-панель:** модерация пользователей, управление справочниками

## Технологический стек

### Frontend

| Технология | Назначение | Обоснование |
|------------|------------|-------------|
| **Vue 3 (Composition API)** | UI фреймворк | Низкий порог входа, реактивность из коробки, быстрая разработка, отличная производительность |
| **TypeScript** | Типизация | Безопасная работа со сложными объектами (груз, маршрут), предотвращение ошибок |
| **Vue Router 4** | Маршрутизация | Официальное решение, защищённые маршруты, вложенные роуты для личных кабинетов |
| **Pinia** | State-менеджмент | Официальное решение Vue, простое API, отличная интеграция с DevTools, поддержка TypeScript |
| **@tanstack/vue-query** | Серверное состояние | Кэширование API-ответов, автоматическое обновление данных, оптимистичные обновления (для ставок), пагинация |
| **VeeValidate + Zod** | Формы и валидация | Простая работа со сложными динамическими формами (создание заказа, управление транспортом). Единые схемы валидации между клиентом и сервером |
| **Axios** | HTTP клиент | Удобная настройка базового URL, перехватчики (interceptors) для автоматической подстановки JWT токенов и централизованной обработки ошибок |
| **Socket.io-client** | Real-time | Новые заказы, уведомления о ставках, обновление статусов |
| **Vite** | Сборка | Быстрая HMR, оптимизированная production-сборка, мгновенный старт разработки |
| **PrimeVUE + Tailwind CSS** | UI компоненты и стилизация | Готовые компоненты (таблицы, модалки, инпуты) + гибкая кастомизация через Tailwind, адаптивный дизайн |

### Backend

| Технология | Назначение | Обоснование |
|------------|------------|-------------|
| **Node.js (NestJS)** | Backend фреймворк | Модульная архитектура, встроенная поддержка WebSockets, единый язык с фронтендом |
| **TypeScript** | Типизация | Общие DTO с фронтендом, строгая типизация, единые схемы Zod |
| **PostgreSQL** | База данных | ACID-транзакции для безопасного бронирования заказов, поддержка PostGIS для геоданных |
| **JWT (Access + Refresh)** | Аутентификация | Безопасное разграничение ролей (shipper / carrier / admin) |
| **Socket.io** | WebSockets | Уведомления, обновление статусов в реальном времени |

### Инфраструктура

| Компонент | Выбор | Обоснование |
|-----------|-------|-------------|
| Frontend хостинг | Vercel / Netlify | Автоматический деплой из GitHub, CDN, поддержка Vue SPA |
| Backend хостинг | Railway.app / Render.com | Простой деплой Node.js, встроенная БД |
| База данных | Supabase (PostgreSQL) | Готовый PostGIS, автобэкапы, удобный клиент, встроенная аутентификация (опционально) |

### Методологии

| Методология | Описание |
|-------------|----------|
| **Feature-based структура** | Организация кода по функциональным модулям (`orders/`, `auth/`, `profile/`, `trucks/`) вместо технических слоёв. Каждый модуль содержит свою логику, компоненты, composables, типы и API-функции |
| **Composables** | Переиспользуемая логика вынесена в composables (useAuth, useOrders, useFilters) для чистоты компонентов |
| **API-стиль** | REST для CRUD + WebSockets для real-time функционала (уведомления о новых ставках и смене статуса заказа) |
| **Безопасность** | Helmet.js, CORS, rate-limiting, валидация DTO (Zod) |

## Визуальное проектирование интерфейса

### Общие принципы UI/UX

Для обеспечения единообразия пользовательского опыта на всех страницах приложения используются унифицированные подходы к отображению состояний:

| Состояние | Компонент | Поведение |
|-----------|-----------|----------|
| **Загрузка (Loading)** | Скелетоны (Skeleton) | Отображаются заглушки, повторяющие структуру контента (карточки, строки таблицы). Используются для первичной загрузки страниц и пагинации |
| **Ошибка (Error)** | Toast + Fallback | Системные ошибки (сеть, 500) отображаются в виде всплывающего уведомления (toast). Критические ошибки на странице сопровождаются кнопкой "Повторить запрос" |
| **Пустое состояние (Empty)** | Иллюстрированная заглушка | Если данных нет (нет заказов, нет транспорта), показывается понятное сообщение и кнопка действия (например, "Создать первый заказ") |
| **Debounce** | Задержка запроса | При вводе текста в поиск или фильтры (геокодер, поиск по заказам) запрос на сервер отправляется с задержкой 300-500мс для снижения нагрузки |

### Страница: Аутентификация (`/auth/*`)

**Тип доступа:** публичный

#### Регистрация (`/auth/register`)

| Шаг | Блок | Описание | API |
|-----|------|----------|-----|
| 1 | Выбор роли | Переключатель "Заказчик" / "Перевозчик" | — |
| 2 | Контакты | Email, телефон, пароль | POST /api/auth/check-email |
| 3 | Данные компании | Название, УНП, контактное лицо | Валидация |
| 4 | (Для перевозчика) | Тип транспорта, регионы работы | GET /api/truck-types |

**Состояния:**
- Валидация в реальном времени (VeeValidate + Zod)
- Отправка: спиннер на кнопке
- Успех: редирект на `/`

#### Вход (`/auth/login`)

| Блок | Описание | API |
|------|----------|-----|
| Форма входа | Email/телефон, пароль, "Запомнить меня" | POST /api/auth/login |
| Доп. ссылки | "Забыли пароль?", "Создать аккаунт" | — |

**Состояния:**
- Неверные данные: сообщение об ошибке
- Блокировка после 5 неудачных попыток
- Успех: редирект на предыдущую страницу

#### Восстановление пароля (`/auth/recover`)

| Шаг | Блок | API |
|-----|------|-----|
| 1 | Ввод email/телефона | POST /api/auth/recover/request |
| 2 | Ввод кода (60с таймер) | POST /api/auth/recover/verify |
| 3 | Новый пароль | POST /api/auth/recover/reset |

**Тип доступа:** публичный

**Основные блоки:**

| № | Блок | Описание | API |
|---|------|----------|-----|
| 1 | Поисковая строка | "Откуда", "Куда", кнопка "Найти" | GET /api/geocode/suggest |

- Изменение фильтра → автоматический перезапрос (Vue Query)

## Личный кабинет перевозчика (`/profile/transporter`)

**Тип доступа:** приватный (только для авторизованных пользователей с ролью `carrier`)

### Структура маршрутов

| Раздел | Маршрут | Доступ | Описание |
|--------|---------|--------|----------|
| Мои ставки | `/profile/transporter/bids` | Приватный | Список всех сделанных ставок с фильтрацией по статусу (активные, принятые, отклонённые, истекшие). Возможность отозвать активную ставку |
| Мои заказы | `/profile/transporter/orders` | Приватный | Список принятых заказов в работе (выполняемые) и архив выполненных. Подтверждение доставки, оставление отзыва о заказчике |
| Управление транспортом | `/profile/transporter/trucks` | Приватный | CRUD операции с транспортными средствами: добавление, редактирование, удаление. Управление статусом доступности ("доступен сейчас") |
| Настройки профиля | `/profile/transporter/settings` | Приватный | Управление личными данными, настройками уведомлений, приватностью, безопасностью и данными компании |

**Управление транспортом:**
- Добавление: тип кузова, грузоподъемность, объем, госномер
- Редактирование, удаление

## Ключевые эндпоинты API

### Аутентификация

| Метод | Эндпоинт | Описание | Тело запроса | Ответ |
|-------|----------|----------|--------------|-------|
| `POST` | `/api/auth/register` | Регистрация нового пользователя | `{ email, phone, password, role, company_name, contact_person }` | `{ user, access_token, refresh_token }` |
| `POST` | `/api/auth/login` | Вход в аккаунт | `{ email, password, remember_me? }` | `{ user, access_token, refresh_token }` |
